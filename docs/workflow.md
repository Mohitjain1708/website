# Workflow

## 1. Positioning

The deck was framed around one proposition: The Dubai Mall is not a mall in the traditional sense, it is a high-traffic commercial platform with retail, luxury, dining, entertainment, and events working together.

## 2. Content Model

The app uses a small content layer instead of hard-coded slides. That keeps the deck easy to extend into deeper sponsor, leasing, and venue pages later.

## 3. Interaction Model

- Sticky chapter navigation for non-linear exploration
- Video-led hero area for immediate impact
- Expandable module cards for future sub-decks
- Smooth scrolling and active section tracking

## 4. Visual Direction

The visual system aims for a luxury-commercial feel:

- dark background
- restrained chrome
- gold and aqua accents
- large editorial typography
- generous spacing and panel contrast

## 5. Performance Approach

- Keep the app to one route for a fast initial load
- Use a small dependency surface
- Lazy-load media where possible
- Favor CSS and SVG-generated surfaces over heavy static assets

## 6. Delivery Checklist

- Local build passes
- README includes setup and design rationale
- Resume bullets included for submission reuse
- Structure is ready for a public GitHub repo and later deployment
